# Pulumi bridge for the Bowtie provider

This directory builds a [Pulumi](https://www.pulumi.com/) provider for Bowtie
from the existing Terraform provider, using
[`pulumi-terraform-bridge`](https://github.com/pulumi/pulumi-terraform-bridge).
The bridge reuses the Terraform provider's schema and Go client, so every Bowtie
resource and data source available in Terraform is also available in Pulumi
across TypeScript, Python, Go, and .NET. There is one implementation to
maintain, not two.

## Why bridge instead of rewrite

The Terraform provider already models the Bowtie API surface (`bowtie_resource`,
`bowtie_policy`, `bowtie_collection`, `bowtie_device_group`,
`bowtie_controller`, `bowtie_org_config`, and the rest). The bridge wraps that
provider as-is:

- New Terraform resources require no bridge code. Pulumi tokens are computed from
  the Terraform resource names (`MustComputeTokens` in `provider/resources.go`),
  so adding a `bowtie_*` resource in Terraform and re-running the generator
  produces a new Pulumi resource in every SDK.
- Terraform fixes (attribute changes, validation, descriptions) are picked up
  the same way. Regenerate; the bridge code does not change.

The one hand-maintained file is `../shim/shim.go` in the Terraform provider
repo. It re-exports the provider constructor from outside the `internal/` tree,
which is the only way a separate Go module can construct the provider. It
changes only if `provider.New()` changes.

A few resources need a one-line entry in the `Resources` map in `resources.go`.
`bowtie_group_membership`, for example, has no `id` attribute in its Terraform
schema, and Pulumi requires every resource to expose an `id`. Its entry derives
the `id` from `groupId`. Token assignment still happens automatically for these
resources; only the `id` is specified by hand.

## Layout

```
shim/shim.go                                       in the Terraform provider repo root
pulumi/
  provider/
    resources.go                                   bridge metadata (ProviderInfo)
    go.mod, go.sum                                 bridge module (replace -> ../../)
    cmd/pulumi-tfgen-bowtie/main.go                schema and SDK generator
    cmd/pulumi-resource-bowtie/main.go             the Pulumi provider plugin
    cmd/pulumi-resource-bowtie/schema.json         generated Pulumi schema (committed)
    cmd/pulumi-resource-bowtie/bridge-metadata.json  generated runtime metadata (committed)
  sdk/{go,nodejs,python,dotnet}                    generated SDK source (committed)
```

`schema.json` and `bridge-metadata.json` are generated by the steps below and
committed so the provider builds without running the generator first. They are
regenerated whenever the Terraform provider changes.

## Prerequisites

- Go 1.22 or newer. The bridge pulls in `pulumi-terraform-bridge/pf`, which
  requires 1.22; recent Go toolchains fetch the right version automatically.
- [Pulumi CLI](https://www.pulumi.com/docs/install/), needed to use or publish
  the provider but not to generate the schema or SDKs.
- The Node, Python, and .NET toolchains, needed only when building those SDKs.

## Regenerate after a Terraform change

Run from `pulumi/provider`. While this module lives inside the Terraform
provider repo, prefix Go commands with `GOWORK=off` so the repo's `go.work`
(which lists only the root module) does not exclude this one. The prefix is
unnecessary once the bridge moves to its own repository.

```bash
# 1. Build the generator and the provider plugin.
GOWORK=off go build ./...

# 2. Generate the Pulumi schema and runtime metadata.
GOWORK=off go run ./cmd/pulumi-tfgen-bowtie schema --out ./cmd/pulumi-resource-bowtie

# 3. Generate language SDKs (the supported targets are go, nodejs, python, dotnet).
GOWORK=off go run ./cmd/pulumi-tfgen-bowtie nodejs --out ../sdk/nodejs
GOWORK=off go run ./cmd/pulumi-tfgen-bowtie python --out ../sdk/python
GOWORK=off go run ./cmd/pulumi-tfgen-bowtie go     --out ../sdk/go
GOWORK=off go run ./cmd/pulumi-tfgen-bowtie dotnet --out ../sdk/dotnet
```

The `replace github.com/bowtieworks/terraform-provider-bowtie => ../../`
directive in `go.mod` points the bridge at the Terraform provider in this
checkout, so a regeneration always reflects the provider's current state.

## Build status

The bridge builds and generates a complete schema. As of the last run against
`pulumi-terraform-bridge/pf v0.49.0` and `v3 v3.96.0`, the generated schema
covers 19 resources and 6 data sources, and all four SDKs (`go`, `nodejs`,
`python`, `dotnet`) generate from it. `go build ./...` and `go vet ./...` are
clean for the bridge, and the generated Go SDK compiles and vets clean. The
generator prints a warning that it cannot fetch upstream documentation; that is
expected until the Terraform provider is published to GitHub, and resource
descriptions still come from the Terraform schema.

## Publishing

The bridge is developed here for now and moves to its own repository before
release.

1. Promote `shim/shim.go` onto the Terraform provider's release branch so it
   ships with the published provider module. It must travel with the Terraform
   provider because it imports `internal/`.
2. Move the `pulumi/` tree into `github.com/bowtieworks/pulumi-bowtie`. Drop the
   `replace` directive and depend on the tagged Terraform provider version.
3. Tag releases. CI runs the generator, builds the four language SDKs, and
   publishes the provider plugin to GitHub Releases, the SDKs to npm, PyPI,
   NuGet, and the Go module proxy, and the package to the Pulumi Registry.
